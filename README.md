# Репозиторий расчета профилей взаимной информации

Этот проект оформляет MATLAB-код расчета профилей взаимной информации в виде чистого репозитория с понятной структурой, единым сценарием запуска и воспроизводимыми выходными файлами.

## Параметры запуска

Основной сценарий работы теперь такой:

1. открыть [run_mi_pipeline.m](run_mi_pipeline.m);
2. изменить переменные в самом верху файла;
3. запустить скрипт целиком.

Ниже перечислены пользовательские переменные, которые можно хардкодить прямо в этом файле.

### Блок путей

- `audio_path`  
  Путь к одному `.wav`-файлу или к папке с `.wav`-файлами.

- `label_path`  
  Путь к папке с Audacity label `.txt`. Если оставить пустую строку `""`, проект работает в режиме без разметки.

- `output_dir`  
  Папка, в которую сохраняются `.mat`, графики и `run_summary.mat`.

### Блок подготовки данных

- `pad_left_sec`  
  Отступ в секундах влево от центра размеченного интервала. Используется в режиме с разметкой.

- `pad_right_sec`  
  Отступ в секундах вправо от центра размеченного интервала. Используется в режиме с разметкой.

- `segment_duration_sec`  
  Длина сегмента в секундах для режима без разметки.

- `segment_overlap_sec`  
  Перекрытие соседних сегментов в секундах для режима без разметки.

### Блок выбора частоты

- `frequency_mode`  
  Режим выбора частоты: `"single"` для одной частоты или `"band"` для диапазона.

- `target_hz`  
  Целевая частота в герцах для режима `"single"`.

- `range_hz`  
  Диапазон частот в виде `[f_low, f_high]` для режима `"band"`.

### Блок бининга

- `binning_type`  
  Тип бининга: `"quantile"` или `"uniform"`.

- `n_bins`  
  Число бинов для дискретизации dB-матрицы.

### Блок MI

- `max_delay_sec`  
  Максимальная задержка для расчета MI в секундах. Профиль строится симметрично: от `-max_delay_sec` до `+max_delay_sec`.

- `window_duration_sec`  
  Длина окна анализа MI в секундах.

- `n_perms`  
  Число перестановок для расчета permutation shift. Если `0`, permutation shift не считается.

### Блок графиков

- `save_segment_preview`  
  Если `true`, дополнительно сохраняется preview сегмента с waveform и спектрограммой.

- `ci_level`  
  Уровень доверительного интервала для графиков. По умолчанию `0.95`, то есть строится 95% доверительный интервал.

### Блок сохранения

- `write_tensor_shards`  
  Если `true`, большие тензоры `mi_values` и `shift_mi` сохраняются в отдельные шард-файлы рядом с основным результатом.

- `max_tensor_shard_size_mb`  
  Целевой максимальный размер одного shard-файла в мегабайтах.

- `tensor_shard_mat_format`  
Формат `save` для shard-файлов. По умолчанию используется `'-v7.3'`.

## Что умеет проект

- читать `.wav`-файлы;
- работать с интервальной разметкой Audacity (`.txt`);
- вырезать аудиосегменты вокруг каждой строки разметки;
- работать и без разметки, автоматически нарезая файл на последовательные сегменты;
- строить спектрограмму и dB-матрицу для дальнейшего расчета MI;
- применять два типа бининга: квантильный и равномерный;
- считать профиль взаимной информации по выбранной частоте или диапазону частот;
- считать permutation shift по исправленному алгоритму;
- сохранять результат расчета и параметры запуска в `.mat`;
- строить графики в стандартизированном виде.

## Требования

- MATLAB R2022 или новее.

## Структура репозитория

```text
run_mi_pipeline.m         - главный запуск пайплайна с параметрами в верхней части файла
examples/                 - готовые примеры запуска
src/                      - основная реализация
  binning/                - модули бининга
  mi/                     - расчет взаимной информации
  plot/                   - стандартизированные графики
  utils/                  - служебные функции
docs/                     - описание структуры проекта
tests/                    - smoke-тесты
results/                  - выходные артефакты локальных запусков
```

Подробности по структуре см. в [docs/project_layout.md](docs/project_layout.md).

## Поддерживаемые режимы работы

### 1. Режим с разметкой

Вход:

- один `.wav`-файл или папка с `.wav`;
- отдельная папка с Audacity label `.txt`;
- имя label-файла должно совпадать с именем аудиофайла.

Формат строки Audacity label:

```text
start_sec<TAB>end_sec<TAB>label_text
```

Для каждой строки разметки создается отдельный сегмент:

- центр сегмента совпадает с центром интервала разметки;
- слева и справа добавляется заданный отступ;
- границы сегмента автоматически ограничиваются границами файла.

### 2. Режим без разметки

Если папка с label-файлами не указана, проект автоматически нарезает аудио на последовательные сегменты фиксированной длины.

Поведение по умолчанию:

- `segment_duration_sec = 30`
- `segment_overlap_sec = 0`

Если файл короче заданной длины сегмента, он обрабатывается как один сегмент.

## Конвейер обработки

1. `prepare_data.m` находит аудио и при необходимости читает разметку.
2. `fft_calc.m` строит спектрограмму.
3. `binning_quantile.m` или `binning_uniform.m` дискретизирует dB-матрицу.
4. `calc_mi_profile.m` считает профиль взаимной информации.
5. `save_mi_result.m` сохраняет `.mat` с метаданными.
6. `plot_mi_standard.m` строит итоговый график.

## Параметры FFT

Параметры FFT задаются внутри файла `src/fft_calc.m` и не пробрасываются из `run_mi_pipeline.m`.

Это сделано специально, чтобы:

- держать настройки FFT в одном месте;
- исключить дрейф параметров между разными сценариями запуска;
- упростить воспроизводимость экспериментов.

## Типы бининга

### Квантильный биннинг

Файл: `src/binning/binning_quantile.m`

- вычисляет квантильные границы отдельно для каждой частотной строки;
- устойчив к неравномерному распределению значений по dB.

### Равномерный биннинг

Файл: `src/binning/binning_uniform.m`

- строит равномерные границы отдельно для каждой частотной строки;
- использует диапазон `min/max` этой строки.

## Расчет MI

Файл: `src/mi/calc_mi_profile.m`

В репозитории реализована только одна каноническая версия расчета MI:

- основана на исправленном `mutual_info_quantile_shift.m`;
- поддерживает permutation shift;
- умеет считать MI как для одной частоты, так и для диапазона частот.

Логика `shift` и `n_perms` находится только в MI-модуле и не смешивается с биннингом.

## Выбор частоты

Поддерживаются два режима:

- одна частота: выбирается ближайший FFT-бин;
- диапазон частот: выбираются все FFT-бины внутри диапазона.

Сырые результаты по частотным бинам сохраняются без потери детализации. Агрегация делается только на этапе графиков.

## Формат результатов

Для каждого сегмента создаются:

- `*_result.mat`
- `*_mi_values_shard_XXXX.mat` при включенном шардировании
- `*_shift_mi_shard_XXXX.mat` при включенном шардировании
- `*_mi_profile.png`
- `*_segment_preview.png` при включенном preview

В `.mat` сохраняются:

- `dataset_meta`
- `fft_result_meta`
- `binning_meta`
- `mi_result`
- `run_timestamp`
- `code_version`
- `run_metadata`

Если включено шардирование, основной `*_result.mat` содержит манифест shard-файлов, а полные тензоры лежат в отдельных `*_mi_values_shard_XXXX.mat` и `*_shift_mi_shard_XXXX.mat`.

## Примеры запуска

### Готовый запуск для большой папки аудио

```matlab
run('run_large_folder_uniform_pm10.m')
```

Этот сценарий подготовлен для режима:

- равномерный биннинг;
- окно MI `1` секунда;
- лаги от `-10` до `+10` секунд;
- сохранение полных тензоров через шардирование на диск.

### Пример без разметки

```matlab
run('examples/example_plain_file.m')
```

Скрипт сам создает небольшой синтетический `.wav` и запускает обработку.

### Пример с разметкой

```matlab
run('examples/example_annotated.m')
```

Скрипт сам создает синтетический `.wav` и соответствующий Audacity `.txt`.

## Использование из своего кода

Минимальный шаблон:

Откройте `run_mi_pipeline.m`, задайте нужные значения в верхнем блоке переменных и запустите файл.

Если в режиме без разметки последний сегмент оказывается слишком коротким для выбранных `max_delay_sec` и `window_duration_sec`, такой сегмент помечается как пропущенный в `run_summary.mat`, а остальная обработка продолжается.


## Тесты

Smoke-тесты лежат в папке `tests/`.

Запуск:

```matlab
runtests('tests')
```

## Что важно помнить

- формат разметки рассчитан на интервальные Audacity labels;
- файл разметки ищется по совпадению базового имени;
- FFT настраивается только в `src/fft_calc.m`;
- permutation shift включается параметром `n_perms`;
- большие записи лучше обрабатывать без целого файла за раз, а сегментами.
