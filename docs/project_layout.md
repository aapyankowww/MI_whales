# Структура проекта

## Корень репозитория

- `run_mi_pipeline.m`  
  Главная пользовательская точка входа. Это скрипт с переменными в верхней части файла.

- `src/run_mi_pipeline_core.m`  
  Внутренняя функция, которая выполняет сам пайплайн по явным аргументам, переданным из верхнего блока переменных пользовательского скрипта.

- `README.md`  
  Основное описание проекта, режимов работы, параметров и примеров запуска.

## Папка `src`

### Подготовка данных

- `prepare_data.m`  
  Определяет режим работы, находит аудио, сопоставляет label-файлы и собирает список сегментов.

- `read_audacity_labels.m`  
  Читает интервальную разметку Audacity из `.txt`.

- `make_segment_from_interval.m`  
  Формирует сегмент вокруг центра размеченного интервала.

- `make_plain_segments.m`  
  Нарезает аудио на последовательные сегменты, если разметки нет.

### FFT и выбор частот

- `fft_calc.m`  
  Строит спектрограмму и dB-матрицу. Внутри этого файла живут параметры FFT по умолчанию.

- `select_frequency_bins.m`  
  Выбирает один FFT-бин по частоте или набор бинов по диапазону частот.

### Сохранение результатов

- `save_mi_result.m`  
  Сохраняет self-contained `.mat` с метаданными запуска и результатом расчета.

## Папка `src/binning`

- `binning_quantile.m`  
  Квантильный биннинг по каждой частотной строке отдельно.

- `binning_uniform.m`  
  Равномерный биннинг по каждой частотной строке отдельно.

## Папка `src/mi`

- `calc_mi_profile.m`  
  Единственная каноническая реализация расчета взаимной информации на основе исправленного `mutual_info_quantile_shift.m`.

## Папка `src/plot`

- `plot_mi_standard.m`  
  Итоговый стандартизированный график MI.

- `plot_segment_preview.m`  
  Preview сегмента: waveform и спектрограмма.

- `export_figure_standard.m`  
  Унифицированный экспорт графиков в PNG.

## Папка `src/utils`

- `ensure_mono.m`  
  Приводит многоканальное аудио к одному каналу.

- `clamp_interval.m`  
  Ограничивает сегмент границами аудиофайла.

- `default_output_dir.m`  
  Формирует стандартную папку вывода для очередного запуска.

## Папка `examples`

- `example_plain_file.m`  
  Самодостаточный пример запуска без разметки.

- `example_annotated.m`  
  Самодостаточный пример запуска с синтетическим Audacity label-файлом.

## Папка `tests`

Содержит smoke-тесты на ключевые контракты:

- чтение разметки;
- plain-сегментацию;
- квантильный биннинг.

## Папка `results`

Локальная папка для выходных `.mat`, `.png` и файлов примеров.  
Если репозиторий загружается на GitHub, эту папку лучше не коммитить как часть исходников экспериментов.
